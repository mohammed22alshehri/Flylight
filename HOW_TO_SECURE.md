# 🔒 كيف تحمي موقعك بالكامل — 4 خطوات

## ⚠️ المشكلة:
صاحبك حمّل الكود وقدر يشتغل لأن قاعدة البيانات **مفتوحة بدون حماية**.

## ✅ الحل: 4 خطوات بسيطة

---

## 🥇 الخطوة 1️⃣: شغّل ملف الحماية في Supabase

### كيف؟
1. ادخل **Supabase Dashboard**
2. من القائمة اليسرى: اضغط **SQL Editor** (أيقونة `</>`)
3. اضغط **+ New query**
4. افتح ملف **`SECURE_DATABASE.sql`** من مشروعك
5. **انسخ كل محتواه** (Ctrl+A → Ctrl+C)
6. الصقه في Supabase (Ctrl+V)
7. اضغط زر **Run** الأخضر
8. تأكد من ظهور `Success` ✅

### ماذا يفعل؟
- 🔐 يمنع قراءة المساهمات بدون كلمة سر
- 🔐 يمنع التعديل/الحذف بدون كلمة سر
- 🔐 يحمي جدول `admin_config`
- 🔐 ينشئ دوال آمنة (RPC functions)

---

## 🥈 الخطوة 2️⃣: تقييد الدومين في Supabase

هذا يمنع أي شخص من استخدام الـ Key من جهازه!

### كيف؟
1. في **Supabase Dashboard**
2. اذهب إلى: **Project Settings** (أيقونة الترس ⚙️)
3. اضغط **API**
4. انزل لقسم **CORS Configuration** أو **Allowed Origins**
5. أضف فقط:
   ```
   https://yourusername.github.io
   https://yourusername.github.io/your-repo-name
   http://localhost:5500 (للتطوير المحلي إذا كنت تستخدمه)
   ```
6. احذف `*` إذا كان موجوداً (هذا يسمح للجميع!)

### النتيجة:
```
صاحبك يحمّل الكود → يشغّله locally
   ↓
المتصفح يحاول الاتصال بـ Supabase
   ↓
Supabase: ❌ "هذا الدومين غير مسموح"
   ↓
الكود يفشل تماماً! ✅
```

---

## 🥉 الخطوة 3️⃣: فعّل Rate Limiting

يمنع الهجمات التي تجرب آلاف الـ passwords:

### كيف؟
1. في **Supabase Dashboard**
2. **Project Settings** → **Auth** → **Rate Limits**
3. اضبط:
   - **Sign-in attempts:** 5 محاولات / دقيقة
   - **API requests:** 100 / دقيقة per IP

### النتيجة:
لو حد جرب يدخل بكلمات سر مختلفة، بعد 5 محاولات يتم حظره ⏱️

---

## 🏆 الخطوة 4️⃣: غيّر كلمة سر الإدارة (الأهم!)

كلمة السر الحالية ربما تكون معروفة. غيّرها فوراً:

### كيف؟
1. في **Supabase SQL Editor**
2. اكتب هذا الكود (غيّر `MY_NEW_STRONG_PASSWORD`):

```sql
-- استخدم كلمة سر قوية: حروف كبيرة وصغيرة وأرقام ورموز
-- مثال: MySecure@Pass123!

update admin_config
set password_hash = extensions.crypt('MY_NEW_STRONG_PASSWORD', extensions.gen_salt('bf', 10))
where id = 1;
```

3. اضغط **Run**
4. تأكد: `1 row updated` ✅

### نصائح لكلمة سر قوية:
- ✅ على الأقل 12 حرف
- ✅ تحتوي حروف كبيرة وصغيرة
- ✅ تحتوي أرقام
- ✅ تحتوي رموز (@#$%&)
- ❌ لا تكون اسمك أو تاريخ ميلادك
- ❌ لا تشاركها مع أحد

# 🎯 بعد كل الخطوات: ماذا يحصل لو صاحبك حاول؟

| محاولة صاحبك | النتيجة |
|---|---|
| يحمّل الكود من GitHub | ✅ يحصل عليه |
| يشغّله على جهازه | ❌ Supabase يرفض (دومين غير مسموح) |
| يغيّر الكود ليتخطى | ❌ RLS يرفض القراءة |
| يحاول يخمّن كلمة السر | ❌ Rate limit يحظره |
| يحاول SQL Injection | ❌ Supabase محمي تلقائياً |
| يحاول يحذف بيانات | ❌ RLS يرفض |
| يحاول يقرأ admin_config | ❌ مغلق تماماً |
